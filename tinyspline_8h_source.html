65 * *
66 * :: Error Handling *
67 * *
68 * The following enums, structs, and macros are used to define and handle *
69 * different types of errors. Use them as follows: *
70 * *
71 * tsStatus status; *
72 * TS_TRY(any_label, status.code, &status) *
73 * // Wrap function calls with TS_CALL. *
74 * TS_CALL(any_label, status.code, ts_bspline_to_beziers( *
75 * &spline, &beziers, &status)) *
76 * if (...) *
77 * // Use one of the TS_THROW macros to raise an error. *
78 * TS_THROW_0(any_label, &status, TS_MALLOC, "out of memory") *
79 * TS_CATCH(status.code) *
80 * // Executed on error. *
81 * printf(status.message); *
82 * TS_FINALLY *
83 * // Executed in any case. *
84 * TS_END_TRY *
85 * *
86 * Although it is always advisable to properly handle errors, embedding your *
87 * code into a TS_TRY/TS_END_TRY block as well as passing a pointer to a *
88 * tsStatus object is entirely optional: *
89 * *
90 * ts_bspline_to_beziers(&spline, &beziers, NULL); *
91 * *
92 * Yet, you may check if a particular function failed, albeit you don't have *
93 * access to the error message: *
94 * *
95 * if (ts_bspline_to_beziers(&spline, &beziers, NULL)) *
96 * // an error occurred *

1259 /******************************************************************************
1260 * *
1261 * :: Transformation functions *
1262 * *
1263 * TinySpline is a library focusing on transformations. That is, most *
1264 * functions are used to transform splines by modifying their state, e.g., *
1265 * their number of control points, their degree, and so on. Accordingly, each *
1266 * transformation functions specifies an input and output parameter (along *
1267 * with the other parameters required to calculate the actual transformation). *
1268 * By passing a different pointer to the output parameter, the transformation *
1269 * result is calculated and stored without changing the state of the input. *
1270 * This is in particular useful when dealing with errors as the original state *
1271 * will never be modified. For instance, let's have a look at the following *
1272 * code snippet: *
1273 * *
1274 * tsBSpline in = ... // an arbitrary spline *
1275 * tsBSpline out; // result of transformation *
1276 * *
1277 * // Subdivide 'in' into sequence of bezier curves and store the result *
1278 * // in 'out'. Does not change 'in' in any way. *
1279 * tsError err = ts_bspline_to_beziers(&in, &out); *
1280 * if (err != TS_SUCCESS) { *
1281 * // fortunately, 'in' has not been changed *
1282 * } *
1283 * *
1284 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
1285 * changed allowing you to handle the error properly. *
1286 * *
1287 * Unless stated otherwise, the order of the parameters for transformation *
1288 * functions is: *
1289 * *
1290 * function(input, [additional_input], output, [additional_output]) *
1291 * *
1292 * 'additional_input' are parameters required to calculate the actual *
1293 * transformation. 'additional_output' are parameters storing further result. *
1294 * *
1295 * Note: None of TinySpline's transformation functions frees the memory of the *
1296 * output parameter. Thus, when using the same output parameter multiple *
1297 * times, make sure to free memory before each call. Otherwise, you will *
1298 * have a bad time with memory leaks: *
1299 * *
1300 * tsBSpline in = ... // an arbitrary spline *
1301 * tsBSpline out; // result of transformations *
1302 * *
1303 * ts_bspline_to_beziers(&in, &out); // first transformation *
1304 * ... // some code *
1305 * ts_bspline_free(&out); // avoid memory leak. *
1306 * ts_bspline_tension(&in, 0.85, &out); // next transformation *
1307 * *
1308 * If you want to modify your input directly without having a separate output, *
1309 * pass it as input and output at once: *
1310 * *
1311 * tsBSpline s = ... // an arbitrary spline *
1312 * tsReal *knots = ... // a knot vector *
1313 * *
1314 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
1315 * *
1316 * Note: If a transformation function fails *and* input != output, all fields *
1317 * of the output parameter are set to 0/NULL. If input == output, your *
1318 * input may have an invalid state in case of errors. *
1319 * *
1320 ******************************************************************************/
